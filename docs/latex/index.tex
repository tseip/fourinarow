Please note that this is a rewrite of Bas van Opheusden\textquotesingle{}s \href{https://github.com/basvanopheusden/fourinarow}{\texttt{ Four-\/in-\/a-\/row implementation}}. Consider him the author of this code for citation purposes.\hypertarget{index_autotoc_md6}{}\doxysection{\texorpdfstring{Dependencies}{Dependencies}}\label{index_autotoc_md6}
This repository requires Python 3 to be installed on your machine.

The build system used by this repo is cmake\+: sudo apt-\/get install cmake

Additionally, this repo requires SWIG to build\+: sudo apt-\/get install swig

If on Windows, install cmake using the Windows installer, and then download and unzip SWIG to a directory. Add the following environment variables for all users\+:

SWIG\+\_\+\+DIR \texorpdfstring{$<$}{<}path to unzipped SWIG directory\texorpdfstring{$>$}{>}

SWIG\+\_\+\+EXECUTABLE \texorpdfstring{$<$}{<}path to swig.\+exe in unzipped SWIG directory\texorpdfstring{$>$}{>}

Finally, add the swig directory (SWIG\+\_\+\+DIR above) to your PATH system variable.\hypertarget{index_autotoc_md7}{}\doxysection{\texorpdfstring{Four-\/in-\/a-\/row build instructions}{Four-in-a-row build instructions}}\label{index_autotoc_md7}
From a fresh checkout, create a build directory\+: ~\newline
 mkdir build ~\newline


Then from the build directory (cd build)\+: ~\newline
 cmake .. ~\newline
 cmake --build . ~\newline
 Note, if on Windows, depending on your Python installation options, you may not have the debug Python libraries installed. If this is the case, specify the Release build at build-\/time\+: cmake --build . --config Release

To run tests, simply execute ./tests in the build output. ~\newline


To install the required Python packages, from the model\+\_\+fitting directory run\+: ~\newline
 pip install -\/r requirements.\+txt ~\newline


To fit a model, from the model\+\_\+fitting directory run\+: python model\+\_\+fit.\+py $<$path\+\_\+to\+\_\+game\+\_\+csv$>$ ~\newline
